# 任务目标：开发基于 Rust + Slint 的 Windows 高级局域网 DHCP 诊断工具

## 1. 项目概述
你需要作为一名高级 Rust 工程师，使用 `Slint` 框架编写一个**仅限 Windows 平台**的桌面 GUI 应用程序。该程序不仅需要基础的 DHCP 探测功能，还必须包含“常用网关存活暴破”以及“基于 DHCP 的二层网络环路检测”功能。程序需编译为无控制台黑框的独立 `.exe`。

## 2. Windows 专属优化与依赖包
请在 `src/main.rs` 的最顶部添加 `#![windows_subsystem = "windows"]`，确保运行时不弹出 cmd 窗口。

在 `Cargo.toml` 中配置以下核心依赖：
- **GUI**: `slint` (并在 `build.rs` 中配置编译脚本)
- **网络底层**: `pnet` (Windows 下自动依赖 Npcap/WinPcap，用于构建和发送二层帧)
- **协议处理**: `dhcproto` (用于构建 DHCP Discover/Request 报文及解析响应)
- **随机数**: `rand` (用于生成安全的 DHCP 事务 ID `xid`)
- **辅助库**: `chrono` (用于带时间戳的日志输出)

## 3. UI 界面布局要求 (Slint 声明式 UI)
请创建一个 `ui/app.slint` 文件，采用干净的 Windows 现代极简布局：
1. **全局属性 (Properties)**:
   - `in-out property <[string]> network-interfaces;`
   - `in-out property <string> selected-interface-name;` 
   - `in-out property <string> current-mac-address;` (选中网卡的 MAC 地址显示)
   - `in-out property <string> log-text: "";`
   - `in-out property <bool> is-scanning: false;`
2. **回调函数 (Callbacks)**:
   - `callback start-full-scan();` (触发 DHCP 发现与网关暴破)
   - `callback start-loop-test();` (触发环路检测)
   - `callback clear-logs();`
3. **视觉布局**:
   - **顶部区域**: `ComboBox` 选择网卡。下方用 `Text` 显示该网卡的 MAC 地址。
   - **操作区域**: 包含三个 `Button`：“全面 DHCP 发现”、“网络环路检测”、“清空日志”。
   - **日志区域**: 一个 `TextEdit` (只读，多行，带滚动条)，占据窗口 70% 以上空间。

## 4. 核心架构与多线程 UI 更新 (⚠️ 严禁阻塞 UI)
- `pnet` 的数据包接收 (`rx.next()`) 是阻塞的，**绝对不可在 Slint 主事件流中抓包**。
- 当触发扫描时，克隆 `AppWindow` 的弱引用 (`app.as_weak()`)，并 `std::thread::spawn` 启动后台线程。
- 在后台线程中，所有日志打印必须通过 `slint::invoke_from_event_loop` 发送闭包到主线程，更新 `log-text`。日志格式必须带时间戳，例如 `[19:59:55] 开始 DHCP 服务器检测...`。

## 5. 核心业务逻辑 (分阶段执行)

### 阶段一：纯广播 DHCP 发现 (无需配置目标 IP)
1. 获取选中的网卡，建立 `pnet::datalink::channel`。
2. 构造极端广播包：以太网目标 MAC 为 `ff:ff:ff:ff:ff:ff`，源 IP `0.0.0.0`，目标 IP `255.255.255.255`。
3. Payload 使用 `dhcproto` 构造 `DHCP Discover` (包含随机 `xid`)。
4. 发送 3 次（每次间隔 1 秒），监听所有的 DHCP Offer，提取并记录局域网内所有响应的 DHCP 服务器 IP 和 MAC。

### 阶段二：常用网关暴破测试
1. 在阶段一完成后，无论是否发现服务器，自动开始遍历硬编码的常用路由器 IP 列表（例如：`192.168.0.1, 192.168.1.1, 192.168.2.1, 192.168.10.1, 192.168.100.1, 10.0.0.1` 等）。
2. 针对列表中的每个 IP，尝试发送定向的 DHCP 探测包。
3. 打印类似日志：`[20:00:02] 检查 192.168.0.1 是否在线...` / `未响应 DHCP 请求`。

### 阶段三：真正的二层网络环路检测 (⚠️ 避坑指南)
当用户点击“网络环路检测”时，执行以下严谨的防误报逻辑：
1. **生成唯一凭证**: 使用 `rand` 生成一个**全新且唯一**的 `xid`（事务 ID）。
2. **单次发包**: 仅向局域网广播**一个**带有此 `xid` 的 `DHCP Discover` 报文。绝对不要触发程序内部的超时重传，防止自己干扰自己。
3. **高频监听**: 持续监听 5 秒。
4. **环路判定算法**: 如果在这个 5 秒内，收到了 **2 个或以上** 带有该特定 `xid` 的 `DHCP Offer` 报文，并且它们到达的**时间差小于 100 毫秒 (0.1秒)**，则断定底层存在二层物理环路（广播风暴复制了数据包）。
5. 打印判定结果和时间差证据到日志区。

## 6. Windows 权限与异常处理
如果 `pnet::datalink::channel` 创建失败（报 Access Denied 或找不到网卡），通过 `invoke_from_event_loop` 将红色的错误信息追加到日志中：“创建底层网络接口失败，请右键选择【以管理员身份运行】此程序，并确保系统已安装 Npcap 环境”。
